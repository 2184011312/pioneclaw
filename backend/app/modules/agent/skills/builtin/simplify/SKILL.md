# 代码审查流程

对 PioneClaw 项目的最近变更或指定文件进行全面的代码审查。

## 项目上下文

PioneClaw 是一个 AI Agent 平台：
- **后端**: FastAPI + SQLAlchemy 2.0 (async) + SQLite，位于 `backend/`
- **前端**: Vue 3 + TypeScript + Element Plus，位于 `frontend/`
- **插件**: 位于 `plugins/`，每个插件一个目录
- **技能**: 文件系统技能位于 `skills/`，内置技能位于 `backend/app/modules/agent/skills/builtin/`

## 审查维度

### 1. 代码复用（DRY）

- 是否存在重复的代码块（≥3 行相似逻辑）？
- 是否有可以抽取为公共函数/工具类的代码？
- 是否重复实现了已有库/框架提供的功能？
- 硬编码的值是否应该提取为常量或配置？
- 前端组件是否可拆分为更小的可复用组件？

### 2. 代码质量

- **命名**：变量、函数、类名是否清晰表达意图？
- **结构**：函数是否过长（>50 行）？嵌套是否过深（>4 层）？
- **职责**：是否存在违反单一职责原则的类/函数？API 路由是否过重？
- **错误处理**：异常是否正确捕获和处理？是否有吞异常的情况？
- **边界条件**：空值、空数组、零值等情况是否处理？
- **类型安全**：Python 类型注解、TypeScript 类型是否完整？

### 3. 性能效率

- 是否存在 N+1 查询问题？（SQLAlchemy async 查询）
- 循环内是否有不必要的重复计算？
- 大数据量操作是否有分页/流式处理？
- 是否有不必要的内存分配或拷贝？
- 异步操作是否正确使用？（async/await 不阻塞事件循环）

## 审查流程

1. 确定审查范围（用户指定，或默认最近变更的文件）
2. 读取所有需要审查的文件
3. 按三个维度分别检查，记录发现的问题
4. 按严重程度排序：
   - **严重**：安全漏洞、数据丢失风险、逻辑错误
   - **中等**：性能问题、可维护性问题
   - **轻微**：命名不规范、样式问题
5. 生成审查报告

## 审查报告格式

```
## 代码审查报告

### 审查范围
- backend/app/api/xxx.py
- frontend/src/views/Xxx.vue

### 发现的问题

#### 严重
- [文件:行号] 问题描述 → 建议修复方案

#### 中等
- [文件:行号] 问题描述 → 建议修复方案

#### 轻微
- [文件:行号] 问题描述 → 建议修复方案

### 总体评价
[1-2句话的总体评价]
```

## 重要原则

- 只审查、不改代码（除非用户明确要求修复）
- 每个问题必须附带具体的文件名和行号
- 每个问题必须给出修复建议
- 不审查第三方库或生成的代码
- 后端 Python 和前端 Vue/TS 各有不同规范，注意区分
