# 定时循环

使用定时任务系统创建周期性执行的自动化任务。

## 工作流程

### 1. 解析用户需求

从用户指令中提取：
- **时间间隔**：如"每5分钟"、"每小时"、"每天9点"
- **执行操作**：要自动执行的任务描述
- **是否持久化**：任务是否需要跨会话保留

### 2. 构建 Cron 表达式

将时间间隔转换为标准 5 字段 cron 表达式：

| 用户表达 | Cron 表达式 | 说明 |
|----------|-------------|------|
| 每5分钟 | `*/5 * * * *` | 每5分钟 |
| 每30分钟 | `*/30 * * * *` | 每30分钟 |
| 每小时 | `7 * * * *` | 每小时（偏移避免整点拥挤） |
| 每2小时 | `7 */2 * * *` | 每2小时 |
| 每天9点 | `57 8 * * *` | 每天8:57（避免整点） |
| 工作日9点 | `57 8 * * 1-5` | 周一至周五 |
| 每周一 | `57 8 * * 1` | 每周一早上 |

**注意**：使用非整点/半点的分钟值（如57、3、13），避免所有用户在同一时刻触发。

### 3. 构建提示词

将用户要执行的操作构建为清晰的任务提示词。提示词应该：
- 完整、独立（不依赖当前对话上下文）
- 包含具体的操作指令
- 说明期望的输出形式

### 4. 创建定时任务

使用 `cron_create` 工具：

```
cron_create(
    cron="*/5 * * * *",
    prompt="检查服务健康状态并报告",
    recurring=true,
    durable=false
)
```

参数说明：
- `cron`：cron 表达式
- `prompt`：每次触发时执行的任务提示词
- `recurring`：true=循环执行，false=一次性
- `durable`：true=持久化（跨会话），false=仅当前会话

### 5. 返回结果

向用户报告：
- 任务 ID
- Cron 表达式
- 下次执行时间
- 是否为持久化任务

### 6. 管理任务

用户可能需要：
- 查看当前任务：使用 `cron_list`
- 删除任务：使用 `cron_delete <task_id>`

## 注意事项

- 默认使用 `durable=false`（会话级任务），除非用户明确要求持久化
- 循环任务有 7 天自动过期限制，如需要长期运行需提醒用户
- 提示词必须独立完整，因为触发时没有当前对话的上下文
- 避免创建过于频繁的任务（<1分钟间隔），可能影响系统性能
